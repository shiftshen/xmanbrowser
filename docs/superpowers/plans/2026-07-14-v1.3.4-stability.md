# XmanBrowser v1.3.4 Stability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make long-running profile browsers self-monitoring and keep long profile content inside stable cards.

**Architecture:** The runner pumps the synchronous Playwright transport and writes a PID-scoped heartbeat. The manager preserves legacy PID checks but terminates current-version runners whose heartbeat stalls. CSS containment handles variable-length profile data without changing stored content.

**Tech Stack:** Python 3.12, pytest, React 19, TypeScript, CSS, Node test runner, Tauri 2/Rust

---
