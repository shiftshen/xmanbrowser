# XmanBrowser v1.3.4 Stability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make long-running profile browsers self-monitoring and keep profile information usable across long content and smaller laptop screens.

**Architecture:** The runner pumps the synchronous Playwright transport and writes a throttled PID-and-token-scoped heartbeat. The manager preserves legacy PID checks but terminates a stalled current-version runner only after matching its per-launch token against the real process command. CSS containment and responsive breakpoints handle variable-length profile data and smaller laptop screens without changing stored content.

**Tech Stack:** Python 3.12, pytest, React 19, TypeScript, CSS, Node test runner, Tauri 2/Rust
